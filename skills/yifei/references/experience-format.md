# 经验文件格式

经验存于用户配置目录 `~/.config/yifei-cli/memory/*.md`（Windows: `%AppData%\yifei-cli\memory\`），**一表/一主题一文件**，私有、不进 git。由 Agent 按本格式直接读写；召回用 `scripts/recall.mjs`。

## 三层权威（冲突时高层优先）
1. **用户决策 (ADR)** — 用户拍板的业务口径/解释，最高权威，来源标「用户确认」。
2. **实测事实** — 通过查询验证的事实，来源是查询。
3. **内置 dict** — 出厂推断（很多 ⚠️），随二进制内嵌，最低。

## 文件模板
```markdown
---
aliases: [COPTC, 销售订单, 销售订单单头, 单别]
tables: [COPTC, COPTD]
updated: 2026-06-18
---

## 用户决策 (ADR)
- 统计「有效销售额」只算单别 221/225，排除 251 样品单 (2026-06-18 用户确认)

## 单别含义
- `221` = 内销订单 (2026-06-18 验证:抽样 TC004 全为内销客户)

## 字段枚举
- `TC008` 币别实测取值: RMB / USD / EUR (2026-06-18)

## 查询坑
- TC003 是 char(8)，区间过滤直接 `BETWEEN '20220101' AND '20221231'` (2026-06-18)

## 数据范围
- 最早订单 2016-11，最晚 2023-10；2018 前稀疏 (2026-06-18)

## SQL 片段
- 按月销售单数: `SELECT SUBSTRING(TC003,1,6) 年月, COUNT(*) FROM COPTC GROUP BY SUBSTRING(TC003,1,6)` (2026-06-18)
```

## 约定
- `aliases` 用于 recall 匹配（表名 + 中文名 + 高频关键词）；`tables` 仅信息性。
- 六个分类标题固定:`用户决策 (ADR)` / `单别含义` / `字段枚举` / `查询坑` / `数据范围` / `SQL 片段`；某类为空则省略该标题。
- 每条事实**带日期**；ADR 类来源标「用户确认」。
- **只写已验证事实 / 用户确认口径，不写猜测**（这是与出厂 dict 的关键区别）。
- 经验与实测冲突 → 更新经验文件那条（标新日期），不盲信旧经验。
